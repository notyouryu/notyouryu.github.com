搭建此博客的心得体会

2022-04-03

一、框架

使用的是 [Hexo](https://hexo.io/)框架。Hexo是一个快速、简洁且高效的博客框架。
特点：
    •速度快
    •可以一键部署
    •插件和可拓展性
    •支持MarkDown

二、安装配件并设置环境变量

依次安装[git](https://git-scm.com/)、[nodejs](http://nodejs.cn/),随后看到绑定域名就ok。
设置环境变量可以参考[我的CSDN](https://blog.csdn.net/apple_54125764/article/details/123919700?spm=1001.2014.3001.5501)。

三、购买域名

域名购买起来其实挺简单的，我选择的是外网的namesilo。在这里踩了坑，不建议选择外网！外网便宜但是最后双向绑定到Hithub的时候会巨慢，内网挺贵、需要公安备案、但是相应速度很快。这里需要注意有的是首年便宜后续特贵，一定要看清楚了。

四、域名与Hithub绑定

在这里踩了巨量的坑，跟着很多文章走下来都没有结果。随后自己摸爬滚打＋求助各路神仙，慢慢绑定成功了。
1、首先进入[Hitub](https://github.com/)界面，注册账号。点击右上角的+，创建一个New repository，选择 public。
github项目创建完成，点击     “Continue to layouts”，点击发布个人网站，我们的初始网站就建好了，不过这只是一个github.io结尾的。
2、安装Hexo，具体步骤如下：
打开git bash 输入以下命令：
•npm install hexo-cli -g
•hexo init blog
•cd blog
•npm install
•hexo server
随后会出来一个[本地连接](http://localhost:4000/),复制到浏览器里我们的本地网站就部署好了，这里有很多[theme](https://hexo.io/themes/)，但是大众化的太多了，我选择的是简约清爽的ayer主题。也可以自己写，当然前提是会css。js的话。
3、部署代码到Hithub page

•首先要生产SSH密钥，这里还是参考CSDN.
•接下来就是将本地的静态博客部署到gitHub上，要把本地的静态博客同步到 Github，我们还需要先安装两个跟部署相关的 hexo 插件。
可以实现如下命令：
• npm install hexo -server –save
• npm install hexo-deployer-git –save
•编辑全局 hexo 的配置文件：_config.yml.编辑全局配置后我们需要重新部署：
• hexo clean #先清除掉已经生成的旧文件
• hexo generate #再生成一次静态文件
• hexo server #进行本地预览，本地预览没问题的情况下，按下ctrl+c取消预览
• hexo deploy #开始部署到Hithub
•访问服务器地址来检查是否部署成功（服务器地址是你的hithub建立的项目地址）。

 小结：以后每次发布新文章都要按照步骤来：
                                    •   hexo clean
                                        hexo generate
                                         hexo deploy

4、github pages绑定域名
•域名解析额&Github解析,这里就不过多叙述了，多看看文章慢慢踩坑。
踩过的坑：域名解析是需要一个A类，一个CNAME的。A类倒不重要，CANME是最为关键的，TTL设置为3600，也就是1 hour.

五、主题选择

我选择了Ayer，看中了它的简约风。修改.yml文件，替换背景image图片，替换logo。图片可以用jpg也可以用npg文件。这里注意：背景图片最好选用偏深色系的image，不然后期博客title会看不见，hexo没有给相关调色代码，所以title目前只能是白色。这里要十分感谢几位朋友帮我找logo，背景图，帮我调试的（zcn，cy，hx，ksy）。zcn的微博在本页最下方哦！

六、常见问题

• hexo g 失败
 解决方法:title: hexo 注意冒号后面要加空格.
• hexo d 失败
 解决方法：获取SSH密钥然后 输入自己注册Hithub时的用户名和邮箱.
 每次修改完成后 按下Ctrl +shift+p 随后输入命令 npm run bulid 和npm run deploy每次deploy完后去hithub中都会有记录 .   
  • 404 not found
 解决办法：GitHub中的https上打勾.
